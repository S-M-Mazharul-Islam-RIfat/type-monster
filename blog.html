Frequently Asked Questions

Some frequently asked questions in interviews which are the basic and most important questions.Everybody must know their answer.

1)In Local Storage -
Local storage persists the storage across browser sessions. So even if the domain is opened in a new tab, or the browser is closed and reopened, the storage is saved. There is no expiration.

2)In Session Storage -
Session storage is cleared when the page session ends. A page session lasts as long as the tab or the browser is open. Therefore, when you open a new tab or a new window, this creates a new session and new storage.

1)In Global Scope -
The variables defined outside of any function or curly brackets are known as global variables and have global scope. Global scope means that the variables can be accessed from any part of that program, any function or conditional state can access that variable.

2)In Block Scope -
If some variables are define inside curly brackets {} or inside a specific function then those variables are called local variables The local variables have a very confined scope which is called the local scope.By doing that, the visibility and accessibility of the variable is only allowed within that function or block.

JavaScript has a runtime model based on an event loop, which is responsible for executing the code, collecting and processing events, and executing queued sub-tasks.The event loop monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it.

We will get undefined in the following 8 ways -
1)Variable which is not initialized.
2)Function with no return.
3)Parameter that is not passed will be undefined.
4)If return has nothing on the right side.
5)Property that doesn't exist on an object.
6)Accessing array elements outside of the index range.
7)If we delete an element inside an array.
8)Set a value directly to undefined.
